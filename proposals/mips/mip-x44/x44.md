# MIP-X44: Revert cbETH Market Oracle on Base

## Summary

This proposal reverts the configured oracle for the cbETH market on Moonwell
Base from the OEV wrapper back to the standard Chainlink ETH/USD price feed,
restoring the oracle configuration that was in place prior to MIP-X43.

## Motivation

MIP-X43 (“Activate OEV wrappers for all remaining markets”) introduced an
incorrect oracle configuration for the cbETH market on Base. Specifically, the
OEV wrapper was pointed at the cbETH/ETH exchange rate feed (`cbETHETH_ORACLE`)
rather than the intended composite oracle price feed (`cbETH_COMPOSITE_ORACLE`).

As a result, the cbETH oracle began returning values around ~1.12 (the cbETH/ETH
exchange rate) instead of ~$1,900+ (the ETH/USD price), effectively changing the
cbETH price feed on Moonwell Base.

Accurate oracle pricing is critical for correct collateral valuation,
liquidation behavior, and overall protocol safety.

## Proposal Details

- Revert the cbETH market oracle on Moonwell Base to:
  - `cbETH_COMPOSITE_ORACLE`
- Remove the OEV wrapper configuration for this specific market.

No other markets or oracle configurations are affected by this change.

## Rationale

- Restores a known-good oracle configuration that was previously live in
  production.
- Prevents incorrect asset pricing that could impact borrowing, liquidations, or
  user confidence.
- Limits scope strictly to the affected market.

## Voting Options

- **For** — Revert the cbETH oracle on Base to `cbETH_COMPOSITE_ORACLE`.
- **Against** — Keep the current OEV wrapper oracle configuration.
- **Abstain** — No preference.
