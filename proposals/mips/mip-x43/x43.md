# MIP-X43: Activate OEV Wrappers for Non-Composite Markets

## Summary

This proposal activates Chainlink OEV Wrapper contracts across all remaining
Moonwell non-composite oracle markets on Base and Optimism. Following successful
validation of OEV functionality in MIP-X38, this proposal expands coverage to 21
additional core markets and 2 Morpho isolated markets.

Composite oracle markets are excluded from this proposal and will be addressed
in a subsequent proposal once compatibility is implemented.

Additionally, this proposal updates the OEV fee split on existing and newly
deployed wrappers from **60/40 to 70/30 (protocol/liquidator)** to improve
protocol value capture based on performance observed during the initial rollout.

## Background

MIP-X38 deployed the updated OEV wrapper infrastructure and activated it on
select feeds as a production validation step:

- Base Core — WETH (CHAINLINK_ETH_USD)
- Base Morpho — WELL (CHAINLINK_WELL_USD)
- Optimism Core — WETH (CHAINLINK_ETH_USD)

With the infrastructure validated, the protocol can extend wrapper activation to
remaining markets utilizing standard Chainlink price feeds.

Markets using composite oracle structures are excluded because their oracle
implementations are not yet compatible with the current wrapper architecture.
These markets will be addressed in a future proposal once compatibility support
is introduced.

Data gathered from the initial deployment phase also indicated that the protocol
can sustainably increase its share of OEV revenue. This proposal therefore
includes an update to the fee split to better align incentives and protocol
revenue capture.

## Proposal Details

### Core Markets

- Deploy new **ChainlinkOEVWrapper** contracts for:
  - 14 Base markets
  - 7 Optimism markets
- Replace legacy wrappers where applicable
- Configure each wrapper as the oracle feed via `setFeed()` on ChainlinkOracle
- Whitelist corresponding mTokens on OEVProtocolFeeRedeemer

**Base Core Markets**

DAI, USDC, USDbC, cbETH, AERO, cbBTC, EURC, WELL, USDS, tBTC, VIRTUAL, MORPHO,
cbXRP, MAMO

**Optimism Core Markets**

USDC, USDT, DAI, WBTC, OP, VELO, USDT0

### Isolated Markets

Upgrade the following oracle proxies to the **ChainlinkOEVMorphoWrapper**
implementation:

- CHAINLINK_MAMO_USD_ORACLE_PROXY
- CHAINLINK_stkWELL_USD_ORACLE_PROXY

Execution method:

- `upgradeAndCall()`
- `initializeV2()`

### OEV Fee Split Update

Update the OEV fee split from **60/40 to 70/30 (protocol/liquidator)** by
modifying the `liquidatorFeeBps` parameter from **4000 → 3000** on existing
wrappers deployed in MIP-X38:

- Base — CHAINLINK_ETH_USD_OEV_WRAPPER
- Base Morpho — CHAINLINK_WELL_USD_ORACLE_PROXY
- Optimism — CHAINLINK_ETH_USD_OEV_WRAPPER

All newly deployed wrappers in this proposal will use the updated **70/30
split** at deployment.

## Rationale

Expanding OEV wrapper coverage increases the protocol’s ability to capture
oracle extractable value generated during liquidation events across compatible
markets, strengthening reserve growth and standardizing oracle infrastructure
across deployments.

Excluding composite oracle markets ensures deployment safety while compatibility
work continues, preventing unnecessary integration risk.

Updating the fee split improves protocol revenue capture while maintaining
incentives for liquidators, based on data from the initial rollout phase.

## Voting Options

- **For** — Activate OEV wrappers, perform upgrades, and update the fee split as
  specified
- **Against** — Do not execute the proposed changes
- **Abstain** — No preference or conflict of interest
