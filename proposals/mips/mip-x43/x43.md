# MIP-X43: Activate OEV Wrappers for Non-Composite Markets

## Summary

This proposal activates Chainlink OEV Wrapper contracts across all remaining
Moonwell oracle markets on Base and Optimism. Following successful validation of
OEV functionality in MIP-X38, this proposal expands coverage to 21 additional
core markets and 2 Morpho isolated markets. Markets using composite oracles are
excluded and will be addressed in a subsequent proposal once compatibility is
implemented.

## Background

MIP-X38 deployed the updated OEV wrapper infrastructure and activated it on
select feeds as a production validation step:

- Base Core — WETH (CHAINLINK_ETH_USD)
- Base Morpho — WELL (CHAINLINK_WELL_USD)
- Optimism Core — WETH (CHAINLINK_ETH_USD)

With the infrastructure validated, the protocol can extend wrapper activation to
remaining markets that utilize standard Chainlink price feeds.

## Proposal Details

Core Markets:

- Deploy new ChainlinkOEVWrapper contracts for:
  - 14 Base markets
  - 7 Optimism markets
- Replace legacy wrappers where applicable
- Configure each wrapper as the oracle feed via `setFeed()` on ChainlinkOracle
- Whitelist corresponding mTokens on OEVProtocolFeeRedeemer

Base Core Markets: DAI, USDC, USDbC, cbETH, AERO, cbBTC, EURC, WELL, USDS, tBTC,
VIRTUAL, MORPHO, cbXRP, MAMO

Optimism Core Markets: USDC, USDT, DAI, WBTC, OP, VELO, USDT0

Isolated Markets: Upgrade the following oracle proxies to the
ChainlinkOEVMorphoWrapper implementation:

- CHAINLINK_MAMO_USD_ORACLE_PROXY
- CHAINLINK_stkWELL_USD_ORACLE_PROXY

Execution method:

- `upgradeAndCall()`
- `initializeV2()`

## Rationale

Expanding OEV wrapper coverage will increase the protocol’s ability to capture
oracle extractable value generated during liquidation events across compatible
markets. This contributes to protocol reserve growth while standardizing oracle
infrastructure and removing legacy wrapper implementations.

The unified architecture ensures consistent fee routing through the
OEVProtocolFeeRedeemer and represents the next phase of the OEV rollout strategy
validated in MIP-X38.

## Voting Options

- **For** — Activate Chainlink OEV wrappers across all remaining compatible core
  and isolated markets
- **Against** — Do not proceed with wrapper activation
- **Abstain** — No preference or conflict of interest
