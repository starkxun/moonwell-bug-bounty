# MIP-X43: Activate OEV Wrappers for All Remaining Markets

### **Summary**

This proposal activates **Chainlink OEV Wrapper contracts** across all remaining
Moonwell core and isolated markets on Base and Optimism. Following the
successful validation of OEV functionality on ETH/USD and WELL/USD feeds in
MIP-X38, this proposal extends coverage to all 31 remaining core markets and 2
additional Morpho isolated markets.

### **Background**

MIP-X38 deployed the new OEV wrapper infrastructure and activated it on three
feeds as an initial validation:

- **Base Core:** WETH (CHAINLINK_ETH_USD)
- **Base Morpho:** WELL (CHAINLINK_WELL_USD)
- **Optimism Core:** WETH (CHAINLINK_ETH_USD)

With OEV now validated in production, this proposal expands activation to all
remaining markets. Pre-existing old-style OEV wrappers (deployed prior to
MIP-X38) that lack the new `chainlinkOracle()` field and
`updatePriceEarlyAndLiquidate()` functionality are deprecated and replaced with
new wrappers.

### **Proposal**

**Core Markets (31 total)**

- Deploy new **ChainlinkOEVWrapper** contracts for 19 Base markets and 12
  Optimism markets
- Replace pre-existing old-style wrappers where they exist
- Wire each wrapper as the oracle feed via `setFeed()` on the ChainlinkOracle
- Whitelist all new mTokens on the OEVProtocolFeeRedeemer

**Isolated Markets (2 additional)**

- Upgrade **CHAINLINK_MAMO_USD_ORACLE_PROXY** and
  **CHAINLINK_stkWELL_USD_ORACLE_PROXY** to the new ChainlinkOEVMorphoWrapper
  implementation via `upgradeAndCall()` with `initializeV2()`

**Markets Covered**

Base Core: DAI, USDC, USDbC, cbETH, wstETH, rETH, weETH, AERO, cbBTC, EURC,
wrsETH, WELL, USDS, tBTC, LBTC, VIRTUAL, MORPHO, cbXRP, MAMO

Optimism Core: USDC, USDT, DAI, WBTC, wstETH, cbETH, rETH, weETH, OP, VELO,
wrsETH, USDT0

Base Morpho: MAMO, stkWELL

### **Rationale**

Expanding OEV wrapper coverage to all markets maximizes the protocol's ability
to capture oracle extractable value from liquidations. The unified wrapper
architecture ensures consistent fee routing through the OEVProtocolFeeRedeemer,
strengthening reserve growth across all markets.

### **Voting Options**

- **For:** Activate Chainlink OEV Wrappers across all remaining Moonwell core
  and isolated markets.
- **Against:** Do not expand OEV wrapper coverage.
- **Abstain:** No preference, conflict of interest, or other.
