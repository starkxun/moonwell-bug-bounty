# MIP-X45: stkWELL Contract Upgrades

## Summary

This proposal upgrades the `stkWELL` contracts across Moonbeam, Base, and
Optimism to remove faulty logic and improve snapshot correctness on Moonbeam.
The upgrade is non-disruptive, preserves all existing balances, and has been
validated using OpenZeppelin upgrade tooling.

## Background

The current `stkWELL` implementation on Base and Optimisim includes a faulty
internal function that is no longer required. In addition, the Moonbeam staking
contract relies on block number–based snapshot logic, which can behave
inconsistently under certain conditions. Updated contract versions have been
prepared to address these issues while maintaining compatibility with existing
state.

## Rationale

Approving this upgrade reduces technical risk, improves the correctness of
reward accounting on Moonbeam, and aligns the staking implementation across
chains. The changes are narrowly scoped, do not alter emissions or staking
mechanics, and require no action from existing stkWELL holders.

## Voting Options

- **FOR** – Approve the stkWELL contract upgrades across Moonbeam, Base, and
  Optimism
- **AGAINST** – Do not execute the upgrades
- **ABSTAIN** – No preference
