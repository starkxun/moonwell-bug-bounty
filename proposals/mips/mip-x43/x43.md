# MIP-X43: Activate OEV Wrappers for Non-Composite Markets

### **Summary**

This proposal activates **Chainlink OEV Wrapper contracts** across all remaining
Moonwell non-composite oracle markets on Base and Optimism. Following the
successful validation of OEV functionality on ETH/USD and WELL/USD feeds in
MIP-X38, this proposal extends coverage to 21 additional core markets and 2
Morpho isolated markets. Markets using composite oracles are excluded from this
proposal and will be addressed in a follow-up after wrapper compatibility is
added.

### **Background**

MIP-X38 deployed the new OEV wrapper infrastructure and activated it on three
feeds as an initial validation:

- **Base Core:** WETH (CHAINLINK_ETH_USD)
- **Base Morpho:** WELL (CHAINLINK_WELL_USD)
- **Optimism Core:** WETH (CHAINLINK_ETH_USD)

With OEV now validated in production, this proposal expands activation to all
remaining markets with standard Chainlink price feeds. Pre-existing old-style
OEV wrappers (deployed prior to MIP-X38) that lack the new `chainlinkOracle()`
field and `updatePriceEarlyAndLiquidate()` functionality are deprecated and
replaced with new wrappers.

Composite oracle markets (wstETH, rETH, weETH, wrsETH, LBTC on Base; wstETH,
cbETH, rETH, weETH, wrsETH on Optimism) are deferred to a follow-up proposal as
their oracle contracts do not implement the `latestRound()` interface required
by the current ChainlinkOEVWrapper constructor.

### **Proposal**

**Core Markets (21 total)**

- Deploy new **ChainlinkOEVWrapper** contracts for 14 Base markets and 7
  Optimism markets
- Replace pre-existing old-style wrappers where they exist
- Wire each wrapper as the oracle feed via `setFeed()` on the ChainlinkOracle
- Whitelist all new mTokens on the OEVProtocolFeeRedeemer

**Isolated Markets (2 additional)**

- Upgrade **CHAINLINK_MAMO_USD_ORACLE_PROXY** and
  **CHAINLINK_stkWELL_USD_ORACLE_PROXY** to the new ChainlinkOEVMorphoWrapper
  implementation via `upgradeAndCall()` with `initializeV2()`

**Markets Covered**

Base Core: DAI, USDC, USDbC, cbETH, AERO, cbBTC, EURC, WELL, USDS, tBTC,
VIRTUAL, MORPHO, cbXRP, MAMO

Optimism Core: USDC, USDT, DAI, WBTC, OP, VELO, USDT0

Base Morpho: MAMO, stkWELL

### **Rationale**

Expanding OEV wrapper coverage to all compatible markets maximizes the
protocol's ability to capture oracle extractable value from liquidations. The
unified wrapper architecture ensures consistent fee routing through the
OEVProtocolFeeRedeemer, strengthening reserve growth across all markets.

### **Voting Options**

- **For:** Activate Chainlink OEV Wrappers across all remaining compatible
  Moonwell core and isolated markets.
- **Against:** Do not expand OEV wrapper coverage.
- **Abstain:** No preference, conflict of interest, or other.
