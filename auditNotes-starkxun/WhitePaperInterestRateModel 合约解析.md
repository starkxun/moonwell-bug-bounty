# WhitePaperInterestRateModel 合约解析

这是 Moonwell（Compound V2 fork）的**利率模型合约**，负责根据市场资金使用情况动态计算借款利率和存款利率。

---

## 核心概念：利用率（Utilization Rate）

这是整个利率模型的基础，公式是：

```
利用率 = 借款总量 / (现金 + 借款总量 - 储备金)
```

直观理解：**池子里的钱有多少比例被借走了**。

- 利用率 = 0%：没人借钱，资金全部闲置
- 利用率 = 80%：池子里 80% 的钱都被借走了
- 利用率 = 100%：资金被借空，新存款人无法提款（极危险）

---

## 利率是怎么算出来的？

这个模型是一条**线性曲线**，利用率越高，借款利率越高：

```
借款利率（每秒）= 基础利率 + 斜率 × 利用率
```

画成图就是一条斜线：

```
借款利率
  │                          ↗
  │                       ↗
  │                    ↗
  │─────────────────↗
基础利率──────────────────────→ 利用率
  0%                        100%
```

**设计意图**：当大量资金被借走时，利率自动升高，吸引新的存款进来，同时劝退部分借款人，维持市场平衡。

---

## 年利率 → 每秒利率的换算

合约里最容易让人绕晕的就是这个换算，来逐行拆解：

```solidity
uint public constant timestampsPerYear = 31536000; // 一年的秒数（365天）

baseRatePerTimestamp = baseRatePerYear
    .mul(1e18)   // 先放大，防止整数除法精度丢失
    .div(timestampsPerYear)  // ÷ 一年秒数
    .div(1e18);  // 再缩回来
```

本质上就是：

```
每秒基础利率 = 年化基础利率 / 31,536,000
```

中间乘除 `1e18` 只是为了在 Solidity 整数运算中保住精度，最终结果是一样的。

**举例**：年化基础利率 2%（即 `0.02e18`）

```
每秒利率 = 0.02 / 31,536,000 ≈ 0.000000000634 (6.34e-10)
```

## 两个利率函数

### getBorrowRate（借款利率）

```solidity
borrowRate = ur × multiplierPerTimestamp + baseRatePerTimestamp
```

就是线性公式，利用率高 → 借款贵。

### getSupplyRate（存款利率）

```solidity
supplyRate = ur × borrowRate × (1 - reserveFactor)
```

拆开理解：

| 部分 | 含义 |
|------|------|
| `borrowRate` | 借款人支付的利率 |
| `× ur` | 只有被借出的那部分钱才产生收益 |
| `× (1 - reserveFactor)` | 协议抽成后，剩下的才归存款人 |

**直觉**：存款人的收益 = 借款人付的利息 × 资金被借出的比例 × 协议不抽走的那部分。

---

## 整体流程图

```
用户存款/借款
      ↓
  利用率变化
      ↓
getBorrowRate → 借款人付更多/更少利息
      ↓
getSupplyRate → 存款人收到更多/更少收益
```
