# MIP-B57: Revert cbETH Market Oracle on Base

## Summary

This proposal reverts the configured oracle for the cbETH market on Moonwell
Base from the OEV wrapper back to the standard Chainlink-composite pricing
model, where the cbETH price is derived as:

**Chainlink cbETH/ETH × Chainlink ETH/USD**

This restores the oracle behavior that was in place prior to MIP-X43 and ensures
cbETH is priced correctly relative to ETH.

## Motivation

MIP-X43 (“Activate OEV wrappers for all remaining markets”) introduced an oracle
configuration for the cbETH market on Base. The OEV wrapper was pointed at the
cbETH/ETH exchange rate feed (`cbETHETH_ORACLE`) directly, without composition
with the ETH/USD feed.

As a result, the oracle began returning values around 1.12 (the cbETH/ETH
exchange rate) instead of the correct USD-denominated cbETH price ($2,200),
effectively breaking cbETH pricing on Moonwell Base.

Accurate cbETH pricing requires composition:

- cbETH/ETH reflects the staking premium
- ETH/USD converts that value into USD terms

This proposal corrects the misconfiguration by restoring the composed Chainlink
pricing model used prior to MIP-X43.

## Proposal Details

- Revert the cbETH market oracle on Moonwell Base to the composed Chainlink
  configuration:
  - **Chainlink cbETH/ETH × Chainlink ETH/USD**
- Remove the incorrect OEV wrapper configuration for this market.

No other markets or oracle configurations are affected.

## Rationale

- Restores the correct USD-denominated cbETH price ($2,200), rather than ETH/USD
  alone ($1,970) or the raw cbETH/ETH rate (1.12).
- Matches the oracle behavior that was previously live and functioning
  correctly.
- Prevents mispricing that could impact collateral valuation, liquidations, or
  borrowing behavior.
- Limits scope strictly to the affected market.

## Voting Options

- **For** — Revert the cbETH oracle on Base to the composed Chainlink
  configuration (cbETH/ETH × ETH/USD).
- **Against** — Keep the current OEV wrapper configuration.
- **Abstain** — No preference.
